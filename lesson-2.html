Welcome to the GWC App Project. This is lesson two. This lesson will teach you how to interact with the user in order to gain their input. In lesson one, we learned that a button can be used to trigger an action. In this lesson, we will learn about several other types of user input: Textboxes, and CheckBoxes.

TextBoxes

TextBoxes are used to gather text input from the user. In general, textboxes are used to gather small inputs like words or phrases, where as another input that behaves similarly called a textarea gathers paragraphs.

Adding a TextBox to your HTML

Adding a textbox requires using the input tag. The input tag has 2 attributes that you should get in the habbit of setting. "type" and "name". "type" specifies what type of input it is, i.e. a textbox, or a checkbox. "name" allows HTML to attach things to it like label. For a textbox use the "text" as the type attribute, and pick any name. Make sure to specify an id as well, so that you may select it in javascript. If you'd like more information about other attributes on "input" tags see http://www.w3schools.com/tags/tag_input.asp

<input type="text" name="my-textbox" id="textbox-1"></input> Note: The id and name do not need to match.

Checkboxes

Checkboxes allow the user to select a subset of a set of options. They have two states "checked" and "unchecked". To create a checkbox, use the "input" tag with the "checkbox" type. Make sure to put the text for the checkbox inside the tags. Also, checkboxes can have their own "onclick" events like the button.

<input type="checkbox" id="checkbox-1">My Fancy TextBox</input>

Grabbing the Checkbox State

Grab the state of the checkbox using the "checked" attribute. It is a boolean (true or false), so we should be able to use it in an if statement.

var is_checked = document.getElementById("checkbox-1").checked; if (is_checked) { alert("You checked the box"); } else { alert("You unchecked the box"); } More information on inputs can be found at:http://www.w3schools.com/tags/tag_input.asp
